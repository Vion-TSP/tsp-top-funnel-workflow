Lead Journey: TOFU → C&S Disposition → Category → MOFU → BOFU

This is the bridging logic that connects the test plan to the nurture engine. Two layers: Layer 1 (C&S sequences for 171 existing leads), Layer 2 (TOFU test sequences for 3,000 new contacts).

Clay List

3,000 contacts built, enriched, split into 3 groups of 1,000

→

LinkedIn Ads

Tests A+B get 2-3 weeks warm-up. Test C skips.

→

Outbound

Test A: emails first. Test B+C: calls first. Via Instantly + C&S.

→

C&S Disposition

SDR tags outcome: SE-Soft Interest, SE-GTM, SE-Not Now, SE-Referral, etc.

→

HubSpot Property

Tag mapped to nurture_category (A-G) + tofu_call_outcome

→

Workflow Trigger

Property change fires workflow. Auto-enrols in correct sequence.

→

Category Sequence

A-G specific cadence, content, channel. From Sully's mailbox.

→

Convert or Extend

Books meeting → BOFU. No convert → MOFU extension (30-90d).

Integration Blockers (Must Resolve Before Launch)

1.
HubSpot Sales Enterprise upgrade — ~£94.50/seat (+£35/seat). Required for trigger-based workflow → sequence enrolment. Without this, all MOFU category routing is manual (max 50 contacts at a time). Approved in principle, not yet purchased.
2.
SDR tagging SOP — SDRs need clear instructions on tagging call outcomes into HubSpot using the new disposition framework (A-G). Without this, leads don't flow into the right category sequences. Needs writing + SDR briefing session.
3.
Instantly → HubSpot reply flow — 986 unread replies sitting in Instantly. Need webhook setup or manual process to push reply/interested signals into HubSpot. Hard blocker for Test A attribution.
4.
Clay list email coverage — Toku Moku list has 2,988 contacts but 0% email coverage. Tests A & B can't run without verified emails. Enrichment needed before launch.
